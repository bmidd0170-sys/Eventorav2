## Architecture

- **Next.js 16** App Router, React 19, TypeScript strict, `@/*` → project root
- **Tailwind v4** via `@tailwindcss/postcss` + `tw-animate-css`. No `tailwind.config.js` — all theme in `app/globals.css` using `@theme inline`
- **shadcn/ui** (New York style, RSC, CSS vars, lucide icons). Add: `npx shadcn@latest add <name>`
- **Dark-only**: `<html className="dark">` hardcoded in root layout (`app/layout.tsx:46`)
- **Components**: `@/components/{ui,landing,app,auth,legal,theme-provider.tsx}`
- **Auth**: Firebase Auth via `@/lib/firebase.ts` (exports `auth`, `db`). `.env` has `NEXT_PUBLIC_FIREBASE_*` vars and `DATABASE_URL` (Neon/Postgres, no ORM installed)
- **Editor API**: `app/api/editor/`

## Security

- **`.env` is tracked in git** — `.gitignore` has `.env*` but the file is already committed. Contains live `DATABASE_URL`, `OPEN_API_KEY`, and Firebase keys. Do not push to public remotes without rotating secrets first.
- `DATABASE_URL` and `OPEN_API_KEY` are server-side only — do not expose in client code.

## Gotchas

- `next.config.mjs` ignores TS build errors. Always run `npx tsc --noEmit` for type checking.
- Generated by v0.app — boilerplate may follow v0 conventions.
- Vercel Analytics renders only in `production`.
- Settings page at `/(app)/settings` contains a Profile tab (this is the profile page).
- tsconfig `target` was `ES6` — now `ES2017`. Revert if you see async/await type errors.
